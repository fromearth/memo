スマホの解像度違いによる見え方をできるだけ統一する
今後出てくる機種は、より高解像度のものが考えられるので画面全体が小さく表示されてしまう問題を回避する為の対策

■meta target-densitydpi の指定
※2011年1月時点でのiPhoneでは対応しておらず、Android2.0以上のみに効果があるプロパティとなる。
<meta name="viewport" content="target-densitydpi=device-dpi,width=device-width,initial-scale=1.0,user-scalable=yes">

■メディアクエリを使ったデバイス・ピクセル比ごとの指定
-webkit-device-pixel-ratio ・・・指定値のデバイスにのみにマッチ
-webkit-min-device-pixel-ratio　・・・指定値以上のデバイスにマッチ
日本のキャリアが販売している多くのandroid端末はこのdevicePixelRatioの値が1.5になっています。(2011年1月現在)

@media only screen and (-webkit-device-pixel-ratio: 1.5),
 only screen and (device-pixel-ratio: 1.5) { /* 多くのAndroid端末 */
    zoom: 150%;
}
@media only screen and (-webkit-device-pixel-ratio: 2),
 only screen and (device-pixel-ratio: 2) { /* iPhone4, iPhone5 */
    zoom: 200%;
}
@media only screen and (-webkit-min-device-pixel-ratio: 3),
 only screen and (min-device-pixel-ratio: 3) { /* これ以上の超高解像度はとりあえずここでまとめて３倍拡大表示 */
    zoom: 300%;
}

■その他CSSの指定
body {
    min-width: 320px;  /* 少なくとも幅 320px は確保する */
    line-height: 1.4;  /* zoomでの縮尺で行間が保たれるように */
    -webkit-text-size-adjust: 100%; /* 文字サイズの自動調整をさせない */
}

※initial-scale または maximum-scale を指定しておくことで、Androidでも -webkit-text-size-adjust が効くらしい。
