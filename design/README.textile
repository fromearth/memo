h1. Webシステム開発設計

h2. 概要

すべての開発は「こういうWebシステムを作りたい」という要望から始まる。
要望は「○○の機能を付けたい」「○○の動きを実現したい」「○○の手間を省きたい」みたいに箇条書きでメモしておく。
このメモに基づいて「要件定義書」を作成する。

基本設計はお客さんに理解してもらう為の文書
詳細設計はプログラマが仕様確認する為の文書

可視化
～詳細設計を上流工程、コーディング以下を下流工程

h2. 要件定義（要求分析フェーズ）

* 要求概要
* システムの目的
** システム化すべきか否かを曖昧にしないためにも定義しておく
* 現状の課題と改善案
** 先の目的がはっきりすれば、現状の足りない課題が見えてくる
** 改善策には、運用で解決するのがスマートなものと、システムで解決するのがスマートな両面がある。
* 基本要件と優先順位
** 要件を「必須」「必要と思われる」「あれば便利」などで分ける。第一フェーズでは「必須」に絞ったシステム開発を行う。
* システム化の範囲
優先順位に沿って、システム化する範囲を定める。ニーズが強く実現性の高いものをやることとして記述する。
やらないことも記述する。（やらない理由も書く）
* 体制図
* スケジュール


h2. 基本設計（外部設計フェーズ）

* 業務フロー
* システム構成図
* 機能一覧表
* 設計書記述様式
* 基本設計書
** 機能ごとに「概要」「Ｉ／Ｏ関連図」「画面／帳票レイアウト」を作成
* ＥＲ図
* テーブル定義書

【流れ】
# 業務フローにシステム対象となる画面や帳票を位置づける。
# 画面や帳票をサブシステム単位に整理して「機能一覧表」を作成する。
# １つずつの機能に対して「基本設計書」を作成。どのデータを使ってどう処理するかをＩ／Ｏ関連図で定義し、具体的な画面・帳票イメージを「画面レイアウト」「帳票レイアウト」で表す。
# 画面・帳票に表示するデータを考慮したテーブル設計を行う。
